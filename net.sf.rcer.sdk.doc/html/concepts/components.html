Component Overview

The SAP R/3 Connector consists of a number of plug-ins and features with a complex set of dependencies. This chapter provides an overview of the components and gives some usage examples.

The connector consists of runtime components and development tools. The runtime components are typically deployed along with any application created using the connector, while the development tools are built on top of these runtime components and provide additional functions, but also require more dependencies to be present e. g. the JDT or oAW). The runtime components depend on the SAP Java Connector (JCo) plug-ins to be present, and since the development tools depend on the runtime components, the same restriction applies to them. To help creating these JCo plug-ins, there's a third component - the JCo Import Wizard - that is completely independent of the other plug-ins and can be used to "bootstrap" the installation process.

JCo Import Wizard

The import wizard consists of two plug-ins and a feature. The feature net.sf.rcer.import is required to turn the import wizard into a separate deployment unit that can be installed from the update site. The import wizard itself is contained in the plug-in net.sf.rcer.jcoimport, its documentation in a separate plug-in net.sf.rcer.jcoimport.doc. The latter depends on a plug-in named net.sf.rcer.doc that provides a common structure for the documentation of the entire connector.

Runtime Components

The runtime components contain functions like connection management, tracing and some tools like the TableReader. The runtime components are distributed into the plug-ins net.sf.rcer.conn (for the UI-independent parts) and net.sf.rcer.conn.ui (for the parts that require the UI to be present). The runtime components should also work without UI access, but that has not been extensively tested yet.

The documentation of the runtime components is contained in the plug-in net.sf.rcer.conn.doc that also depends on net.sf.rcer.doc.

The runtime components are not contained in a special feature.

Development Tools

The development tools are bundled in a feature named net.sf.rcer.sdk. This feature also contains the runtime components, the examples and the generated source plug-ins.

The RFC mapping language is defined in net.sf.rcer.rfcgen; the editor and generator are contained in net.sf.rcer.rfcgen.editor and net.sf.rcer.rfcgen.generatory respectively.

The documentation is distributed among several plug-ins: net.sf.rcer.sdk.doc for the general topics, net.sf.rcer.rfcgen.doc for the RFC mapping language. These plug-ins each depend on a plug-in named net.sf.rcer.doc that provides a common structure for the documentation of the entire connector.

The development tools also contain an experimental repository object model that is contained in net.sf.rcer.rom.

Examples

The examples are contained in several plug-ins that form separate RCP applications:

  • net.sf.rcer.example.rfcgen.pojo.call
  • net.sf.rcer.example.rfcgen.pojo.rr
  • net.sf.rcer.example.traditional

Other Plug-Ins

If you start digging into the internals of the connector, you may encounter one of the following plug-ins:

  • net.sf.rcer is a feature that is used for the build process only.
  • net.sf.rcer.conn.test contains the automated tests for net.sf.rcer.conn that do not require a connection to a SAP R/3 system.
  • net.sf.rcer.conn.test-r3 contains the automated tests for net.sf.rcer.conn that require a connection to a SAP R/3 system.
  • net.sf.rcer.rom.test contains the automated tests for net.sf.rcer.rom. These tests require a connection to a SAP R/3 system.
  • net.sf.rcer.test.provider contains the connection providers for the automated tests.
